Версия Java - 17
Система сборки - Maven 3.9.4



Из особенностей реализации - внутри классов для полей с названием Integer я использовал тип Long,
т.к. слово Integer в моем решении больше воспринимается как "целое число", а не тип данных,
а так же из-за того, что Long имеет больший диапазон значений.

В дополнение к предыдущему, проверку на границы диапазона я не делал. Все невалидные значения будут восприниматься
как String.

Logger в данной реализации не используется, вывод идет сразу в консоль.

Сторонние библиотеки не использовал

Файлы для обработки необходимо поместить в папку 'files_to_parse'
Результирующие данные располагаются в папке results

В реализации содержатся 2 тестовых файла для обработки '1.txt' и '2.txt'

Пример команды запуска с несуществующим файлом 'file.txt' и неизвестным параметром - java -jar .\test_task_shift-1.0-SNAPSHOT.jar -i -p words- -f 1.txt 2.txt file.txt
